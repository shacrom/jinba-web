## Compact Rules (inject into sub-agents that touch this repo)

- **SEO es core**: cada página tiene `<title>`, `<meta description>`, Open Graph, Twitter Card, `hreflang` ES/EN. Lighthouse SEO ≥ 95 antes de merge.
- **Core Web Vitals ≥ 90** (LCP, INP, CLS) antes de merge. Islands mínimos, imágenes WebP + `loading="lazy"` + dimensiones explícitas.
- **No cliente-side Supabase** para datos públicos — siempre server-side loader, para SEO y seguridad.
- **Tipos importados desde `jinba-db/types/database.ts`** — no redeclarar schemas.
- **i18n strict**: ningún string hard-coded en componentes; toda copy vive en `src/content/` o diccionarios tipados. Ambas lenguas siempre sincronizadas (CI falla si una versión falta).
- **Imágenes de listings**: consumir el thumbnail ya procesado en Supabase Storage (el scraper garantiza blur + redimensión). Nunca fetch a URL original desde el browser.
- **Accesibilidad**: `alt` obligatorio, landmarks ARIA correctos, navegación por teclado probada en las 3 páginas principales (home, ficha modelo, listados).
- **Claude Design como source of truth** para diseño visual en F1: tokens, spacing, componentes — antes de implementar cualquier UI, verificar contra el diseño aprobado.
